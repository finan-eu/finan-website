## Navigation Responsive Breakpoint Fix

### Problem
The navigation menu appeared cramped and overlapping at 1152px viewport width, causing poor user experience in the mid-range screen size between tablet and desktop.

### Solution
Changed the responsive breakpoint from `lg:` (1024px) to `xl:` (1280px) for the navigation menu. This ensures the mobile hamburger menu is displayed for all screens below 1280px, while the full desktop navigation only appears on screens 1280px and above where there's adequate space.
