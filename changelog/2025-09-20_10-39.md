## Security Enhancement

### Content Security Policy (CSP) Update

**What changed:**
- Updated CSP meta tag in `src/layouts/Layout.astro` to include specific script hash
- Added `sha256-zKHFz8kxOP0VPHX+Zubhc0/i0b20swNFpJoJK8XmHmg=` to `script-src` directive

**Why:**
- Resolves CSP violation warning in local development server
- Maintains security by only allowing specific, whitelisted inline script to execute
- Eliminates console error: "Refused to execute inline script because it violates CSP directive"

**Impact:**
- ✅ No more CSP warnings in local dev environment
- ✅ Security maintained with hash-based script allowlisting
- ✅ Production deployment unaffected (error was dev-only)

**Technical details:**
- CSP hashes are cryptographic fingerprints of exact script content
- More secure than `'unsafe-inline'` as they only allow specific scripts
- Hash exposure in headers is safe and standard security practice
