# Changelog - 2025-09-29 23:56 (Europe/Helsinki)

## Fix: Remove Dynamic Viewport Height from Hero Section

### Problem
The hero section was using `min-h-screen-mobile` class which relied on dynamic viewport height calculations (`100vh`, `100dvh`, and JavaScript-based `--vh` custom property). This caused layout shifting on mobile devices when scrolling, as the browser chrome (address bar, toolbar) would hide/show and trigger viewport height recalculations.

### Solution
Replaced the dynamic viewport height approach with fixed padding:
- Removed: `min-h-screen-mobile` class
- Added: `py-20 md:py-32` (80px mobile, 128px desktop padding)

### Changes Made
- **File Modified**: `src/components/HeroHeader.astro`
  - Line 12: Changed `class="relative min-h-screen-mobile flex items-center py-8"`
  - To: `class="relative flex items-center py-20 md:py-32"`

### Testing
Validated with Playwright MCP on mobile viewport (375x667):
- Hero section height remains stable at 808px
- No layout shifting when scrolling up/down
- Height consistency confirmed across multiple scroll events

### Impact
- Improved mobile user experience
- Eliminated jarring layout shifts during scroll
- Content-based height provides more predictable layout behavior
