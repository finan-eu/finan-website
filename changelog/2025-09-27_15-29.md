# Changelog - 2025-09-27 15:29 (Europe/Helsinki)

## Mobile Viewport Height Fix Implementation

### Problem Fixed
Fixed mobile browser layout shifting issues where the website layout would shift when scrolling on mobile devices as the browser's address bar/toolbar appears and disappears.

### Changes Made

#### 1. CSS Custom Properties Solution (`src/styles/global.css`)
- **Added**: CSS custom properties for mobile viewport height fix
- **Added**: `--vh` custom property initialized to `1vh`
- **Added**: `.min-h-screen-mobile` utility class with progressive enhancement
- **Added**: `.h-screen-mobile` utility class with progressive enhancement
- **Added**: `@supports` rule for modern `dvh` unit detection

**Technical Implementation:**
```css
:root {
  --vh: 1vh;
}

.min-h-screen-mobile {
  min-height: 100vh;                    /* Fallback */
  min-height: calc(var(--vh, 1vh) * 100); /* Custom property */
  min-height: 100dvh;                   /* Modern browsers */
}
```

#### 2. JavaScript Viewport Height Detection (`src/layouts/Layout.astro`)
- **Added**: Inline script for real-time viewport height calculation
- **Added**: Debounced resize event handling (100ms delay)
- **Added**: Orientation change support with proper timing
- **Added**: Page navigation persistence via `pageshow` event
- **Updated**: CSP headers to allow inline scripts (`'unsafe-inline'` added to script-src)

**Key Features:**
- Calculates actual viewport height: `window.innerHeight * 0.01`
- Updates CSS custom property: `--vh` in real-time
- Prevents performance issues with debouncing
- Handles device orientation changes

#### 3. Hero Component Update (`src/components/HeroHeader.astro`)
- **Changed**: `min-h-[100svh]` → `min-h-screen-mobile`
- **Benefit**: Now uses the mobile-safe viewport height class

#### 4. Security Headers Update (`src/layouts/Layout.astro`)
- **Updated**: Content Security Policy to allow inline viewport detection script
- **Added**: `'unsafe-inline'` to script-src directive
