# Changelog - 2025-09-21_02-10

## Content Security Policy (CSP) Fix

### Problem Resolved
- **Issue**: Deployed site showing CSP violation error: "Refused to execute inline script because it violates the following Content Security Policy directive"
- **Root Cause**: Navigation JavaScript was inline in `src/components/Navbar.astro`, violating the CSP `script-src 'self'` directive
- **Impact**: Console errors on production site, potential functionality issues

### Solution Implemented
**Moved inline navigation script to external file for CSP compliance and better maintainability**

#### Changes Made:

1. **Created External Script File**
   - **New**: `public/js/navbar.js`
   - Contains all mobile menu and dropdown functionality
   - Served as static asset from `/js/navbar.js` URL

2. **Updated Navbar Component**
   - **File**: `src/components/Navbar.astro`
   - **Before**: Inline `<script>` tag with navigation JavaScript
   - **After**: `<script is:inline src="/js/navbar.js"></script>`
   - Used `is:inline` directive to prevent Astro bundling

3. **Simplified CSP Policy**
   - **File**: `src/layouts/Layout.astro`
   - **Before**: `script-src 'self' 'sha256-IOY1DIcq9sRs/pZ5Ufa6FVpCtIvL7rnzOsIUlHFEPyg='`
   - **After**: `script-src 'self'`
   - Removed hash requirement for cleaner, more maintainable policy

### Benefits Achieved
- ✅ **Security**: Eliminated CSP violations while maintaining strict security policy
- ✅ **Maintainability**: No need to update script hashes when code changes
- ✅ **Functionality**: All navigation features work correctly (mobile menu, dropdowns)
- ✅ **Performance**: External script can be cached by browsers
- ✅ **Compliance**: Follows security best practices for static sites
