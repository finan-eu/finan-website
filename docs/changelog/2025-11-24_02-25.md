# Changelog - 2025-11-24 02:25 (Europe/Helsinki)

## Fixed

### Content Security Policy (CSP) Configuration for Ghost Blog Images

- **Fixed CSP blocking external images**: Updated Content Security Policy to allow Ghost blog post images from external CDNs
  - Modified `src/layouts/Layout.astro` (line 64) to update CSP meta tag
  - Added Vite middleware plugin in `astro.config.mjs` for development server headers
  - Updated `security.config.js` for consistency across configurations

#### CSP Image Sources Added

The `img-src` directive now includes:

- `https://images.unsplash.com` - Unsplash images used in Ghost blog posts
- `https://*.ghost.io` - Ghost Pro hosted images
- `https://*.ghost.org` - Ghost.org CDN images
- `https://*.ghostcdn.com` - Ghost CDN
- `https://*.cloudfront.net` - CloudFront CDN (used by some Ghost instances)

#### Technical Details

**Before**: CSP blocked all external images with `img-src 'self' data:`, causing Ghost blog post thumbnails to fail loading with CSP violations.

**After**: Ghost blog images from Unsplash and Ghost CDN now load correctly while maintaining security through allowlist-based CSP.

**Files Modified**:

1. `src/layouts/Layout.astro` - Updated CSP meta tag in HTML head
2. `astro.config.mjs` - Added custom Vite plugin with middleware for dev server CSP headers
3. `security.config.js` - Updated security configuration for consistency
