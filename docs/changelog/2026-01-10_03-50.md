# Changelog - 2026-01-10 03:50 (Europe/Helsinki)

## Added

### Registration Modal System
- **New Component: `Modal.astro`** - Reusable modal component with comprehensive accessibility features
  - WCAG 2.1 compliant focus trap that contains keyboard navigation within modal
  - Full keyboard support (Escape to close, Tab navigation with focus trap)
  - iOS Safari scroll lock with proper scroll position restoration
  - Accessible ARIA attributes (role="dialog", aria-modal, aria-labelledby)
  - Click-outside-to-close functionality with overlay
  - Smooth entrance/exit animations
  - Memory leak prevention with proper event listener cleanup on navigation

- **New Component: `RegistrationModal.astro`** - Event-specific registration modal
  - Three Stripe payment category options (FiNAN Members, Non-Members, Speakers)
  - Integrated payment links for Triennial Gathering 2026
  - Consistent button styling with hover effects and focus indicators

- **Modal Integration in `EventHeaderSection.astro`**
  - "Register Now" button now triggers modal instead of anchor link navigation
  - Comprehensive error handling and validation for modal trigger connection
  - TypeScript interface for type-safe modal API

## Fixed

### Critical Accessibility Issues
- **Focus Trap Implementation** - Added WCAG 2.1 compliant focus trap to prevent keyboard users from tabbing out of modal
- **Keyboard Navigation** - Enhanced keyboard accessibility with proper focus management and focus-visible styles
- **Focus Restoration** - Modal now properly returns focus to trigger button when closed

### Critical Error Handling
- **Memory Leak Prevention** - Fixed memory leak from orphaned event listeners by implementing cleanup on `astro:before-swap`
- **parseInt Validation** - Added NaN validation and error handling for scroll position restoration to prevent silent failures
- **DOM Element Validation** - Added comprehensive validation for all DOM queries with descriptive error logging:
  - Modal close button and overlay existence checks
  - Countdown timer element validation
  - Modal trigger connection validation
- **Countdown Timer Safety** - Replaced unsafe non-null assertions (`!`) with proper null checks and try-catch blocks

### Type Safety
- **TypeScript Interfaces** - Replaced all unsafe `as any` type assertions with proper `ModalElement` interface
- **Type-Safe Modal API** - Defined clear TypeScript interface for `_openModal` method

## Changed

### Code Quality Improvements
- **Enhanced Error Logging** - All errors now include descriptive context with component prefixes (`[Modal]`, `[EventHeader]`)
