# Changelog - 2025-12-10 01:43

## Added

- **Multi-tag support for blog representation**: Updated the blog representation configuration to support filtering posts by multiple tags. The `BlogRepresentationConfig` interface now accepts both single tags (`string`) and arrays of tags (`string[]`).

- **Kingdom of Denmark multi-tag configuration**: Configured the Kingdom of Denmark representation page to display blog posts tagged with any of: "Kingdom of Denmark", "Denmark", "Faroe Islands", or "Greenland".

## Changed

- **Ghost API implementation with in-memory filtering**: Refactored `getBlogPostsByTag()` function in `src/lib/ghost.ts` to use in-memory filtering instead of API-level filtering. This workaround addresses TypeScript validation limitations in the `@ts-ghost/content-api` library that doesn't support tag filters in browse parameters.
  - Now fetches all posts with `limit: 'all'`
  - Filters posts client-side by matching tag slugs
  - Supports both single-tag and multi-tag filtering seamlessly

- **Enhanced empty state message**: Updated `BlogRepresentation.astro` to properly display multiple tags in the empty state message using array join logic.

## Fixed

- **Backward compatibility maintained**: Existing single-tag configurations (Finland, Sweden, Norway, etc.) continue to work without any changes required.

## Technical Details
