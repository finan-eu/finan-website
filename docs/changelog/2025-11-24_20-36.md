# Changelog - 2025-11-24 20:36 (Europe/Helsinki)

## Added

### Ghost CMS Content Client Abstraction

- Introduced `src/lib/ghost.ts` centralizing all Ghost Content API access behind lazy-initialized helper functions.
- Provides type-safe wrappers and graceful fallbacks when environment variables are missing.
- Establishes foundation for future News & Events dynamic content.

### Environment Variable Validation for Ghost Integration

- Added explicit checks for `GHOST_URL` and `GHOST_CONTENT_API_KEY` with console warning when absent.
- Prevents runtime errors and ensures UI can render an intentional empty state.

## Changed

### Defensive Fetch Patterns

- Standardized empty array fallbacks across Ghost fetch helpers to avoid undefined handling in consuming components.
- Improves resilience during intermittent API outages.

### Documentation & Inline Clarity

- Expanded JSDoc comments in `ghost.ts` describing lazy initialization strategy and error handling rationale.

## Impact

- Solid baseline for dynamic post retrieval without breaking existing static pages.
- Reduced risk of hard failures if Ghost service or credentials are misconfigured.
- Clearer separation of concerns: UI components consume data; `ghost.ts` owns fetching logic.

## Files Touched

1. `src/lib/ghost.ts` (new abstraction / comments / validation logic)
2. `src/components/Blog.astro` (consumption of defensive fallbacks — no user-facing regressions)

## Validation

- Project builds successfully (`npm run build`).
- Blog component renders empty state gracefully with missing or invalid Ghost configuration.
- No lint errors introduced; formatting consistent with existing style.
