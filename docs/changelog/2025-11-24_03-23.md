# Changelog - 2025-11-24 03:23 (Europe/Helsinki)

## Added

### Ghost CMS Auto-Deploy Documentation

- **Created comprehensive setup guide** for Ghost CMS → Cloudflare Pages webhook integration
- **File**: `docs/GHOST_WEBHOOK_SETUP.md`
- **Purpose**: Enable automatic website rebuilds when Ghost content is published or updated

### Documentation Includes

- Step-by-step instructions for creating Cloudflare Pages deploy hooks
- Ghost webhook configuration for post events (published, updated)
- Testing procedures to verify the integration works
- Troubleshooting section for common issues
- Security best practices for webhook URLs
- Monitoring and maintenance guidelines
- Advanced configuration options for selective rebuilds

### Key Features

- **Auto-deployment workflow**: Ghost posts trigger automatic Cloudflare Pages builds
- **Expected deployment time**: 1-3 minutes from post publish to live site
- **Webhook events supported**: Post published, post updated, post unpublished, post deleted
- **Build monitoring**: Instructions for tracking deployment status and build quota

## Context

This documentation was created in response to the question about whether the Astro website automatically rebuilds when Ghost CMS content changes. The answer was no, so this guide provides the solution to enable automatic deployments through webhooks and deploy hooks.

## Impact

- Eliminates manual rebuild/deploy steps when updating blog content
- Reduces time to publish from content creation to live site
- Improves content workflow for the FiNAN team
- Ensures blog content stays synchronized with Ghost CMS
