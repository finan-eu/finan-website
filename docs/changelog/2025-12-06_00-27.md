# Changelog - 2025-12-06

## Added

### Blog Representation Component
- Created `BlogRepresentation.astro` component for displaying country-specific blog posts on representation pages
- Component supports customizable headings and descriptions per country
- Implements tag-based filtering to show only relevant posts for each Nordic region
- Includes empty state handling when no posts are available
- Maintains consistent design with the main Blog.astro component

### Blog Representation Configuration
- Added `blogRepresentation.ts` data file with TypeScript configuration for all 8 Nordic countries/regions:
  - Finland
  - Sweden
  - Norway
  - Denmark
  - Iceland
  - Faroe Islands
  - Greenland
  - Kingdom of Denmark
- Each configuration includes customizable heading, description, and tag filter
- Fully type-safe with `RepresentationCountry` type export

### Ghost CMS Integration
- Added `getBlogPostsByTag()` function to Ghost API library (src/lib/ghost.ts)
- Fetches and filters blog posts by tag name (case-insensitive)
- Returns up to 6 posts by default (configurable limit)
- Implements client-side filtering to avoid ZodError issues with Ghost API filter parameter

### Representation Page Updates
- Added BlogRepresentation component to all 8 representation country pages:
  - src/pages/representation/finland.astro
  - src/pages/representation/sweden.astro
  - src/pages/representation/norway.astro
  - src/pages/representation/denmark.astro
  - src/pages/representation/iceland.astro
  - src/pages/representation/faroe-islands.astro
  - src/pages/representation/greenland.astro
  - src/pages/representation/kingdom-denmark.astro
- Blog sections positioned before CTA banner for consistent user experience

## Fixed

- Resolved ZodError when filtering Ghost CMS posts by tag by implementing client-side filtering instead of using Ghost API's filter parameter

## Technical Details

- All changes follow project code quality standards (ESLint + Prettier)
- Build successful with no errors (17 pages built in 1.39s)
- Fully type-safe implementation using TypeScript
- Maintains consistent component architecture across the application
