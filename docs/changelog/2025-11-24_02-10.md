# Changelog - 2025-11-24 02:10 (Europe/Helsinki)

## Added

### Ghost CMS Integration

- **Ghost Content API Integration**: Implemented full Ghost CMS integration using `@ts-ghost/content-api` for type-safe content management
  - Added `@ts-ghost/content-api` package (4.2.0) for TypeScript-first Ghost API client
  - Created Ghost API client in `src/lib/ghost.ts` with helper functions:
    - `getBlogPosts(limit)` - Fetch multiple blog posts with customizable limit
    - `getPostBySlug(slug)` - Fetch individual post by slug
    - `getAllPostSlugs()` - Get all post slugs for static path generation

### Blog Components and Pages

- **Updated Blog Component** (`src/components/Blog.astro`):
  - Integrated dynamic Ghost post fetching (displays 6 latest posts)
  - Displays post thumbnail images with fallback to placeholder
  - Shows primary tag, formatted publication date, title, and excerpt
  - "Read More" button links to individual post pages
  - Updated "View All" button to link to `/blog`

- **Blog Listing Page** (`src/pages/blog/index.astro`):
  - Created comprehensive blog archive page
  - Displays all published posts in grid layout
  - Includes PageHeader component for consistent styling
  - Shows empty state message when no posts are available
  - Responsive design with existing Tailwind CSS styles

- **Individual Post Page** (`src/pages/blog/[slug].astro`):
  - Dynamic routing for individual blog posts
  - Displays feature image, tags, title, author, publication date, and excerpt
  - Renders full HTML content from Ghost with custom prose styles
  - "Back to Blog" navigation at top and bottom
  - Custom CSS for Ghost content formatting (headings, blockquotes, code, etc.)
  - 404 redirect for non-existent posts

### Environment and Type Safety

- **Environment Variables**:
  - Updated `.env.example` with Ghost CMS configuration:
    - `GHOST_URL` - Ghost site URL
    - `GHOST_CONTENT_API_KEY` - Content API key for read-only access
  - Added comprehensive documentation comments for obtaining credentials

- **TypeScript Definitions** (`src/env.d.ts`):
  - Created environment type definitions for `GHOST_URL` and `GHOST_CONTENT_API_KEY`
  - Ensures type safety for environment variables throughout the application
