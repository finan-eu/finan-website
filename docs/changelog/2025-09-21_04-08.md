# Changelog - 2025-09-21_04-08

## Mobile Layout Stability Improvements

### Fixed Mobile Viewport Issues

- **Updated `PageHeader.astro`**:
  - Changed `min-h-[50vh]` to `min-h-[50dvh]` to prevent layout shifts when mobile browser address bar appears/disappears during scrolling
  - Ensures consistent header height regardless of mobile browser UI changes

- **Updated `HeroHeader.astro`**:
  - Changed `min-h-screen` to `min-h-[100dvh]` to prevent layout shifts on mobile devices
  - Maintains full-height hero section while adapting to dynamic viewport changes

### Technical Details

The `dvh` (dynamic viewport height) unit adapts to the actual visible viewport height on mobile devices, ignoring browser UI elements that slide in and out during scrolling. This prevents the jarring layout shifts that occur with traditional `vh` units on mobile browsers.
