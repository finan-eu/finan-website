## 🐛 Fix: Mobile Menu Dropdown Issue with View Transitions

### Overview

Resolved a critical issue where mobile navigation dropdown menus stopped functioning after navigating between pages using Astro's view transitions feature.

### Problem Description

- Mobile menu worked correctly on initial page load
- After navigating to any page via view transitions, the chapters dropdown became non-functional
- Issue was caused by JavaScript event listeners being lost during view transitions
- Original implementation relied on `DOMContentLoaded` which doesn't fire on subsequent view transition navigations

### Solution Implemented

#### ✅ Root Cause Analysis

- **Issue**: External JavaScript file `/js/mobile-menu.js` only initialized on `DOMContentLoaded`
- **Impact**: Event listeners were not re-attached after view transitions
- **Scope**: Affected both mobile menu toggle and chapters dropdown functionality

#### ✅ Technical Fix

- **Moved JavaScript inline**: Relocated mobile menu JavaScript from external file into `Navbar.astro` component
- **Added view transition support**: Implemented `astro:after-swap` event listener for re-initialization
- **Memory leak prevention**: Used element cloning technique to properly remove old event listeners
- **Maintained functionality**: All existing features preserved including animations and accessibility

#### 📁 Files Modified

- `src/components/Navbar.astro`
  - Removed reference to external `/js/mobile-menu.js`
  - Added inline `<script>` with `initializeNavbar()` function
  - Implemented dual event listeners: `DOMContentLoaded` and `astro:after-swap`
  - Added proper cleanup logic to prevent memory leaks
