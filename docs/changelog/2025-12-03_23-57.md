# Changelog - 2025-12-03 23:57 (Europe/Helsinki)

## Added

- **Cloudflare Pages Redirect Rules**: Created `_redirects` file for Cloudflare Pages
  - Implements 301 permanent redirects from `finan-website.pages.dev/*` to `finan.eu.com/:splat`
  - Preserves URL paths and query parameters during redirect
  - Prevents users and search engines from accessing the default Cloudflare Pages domain
  - Ensures brand consistency by enforcing custom domain usage

- **Dynamic robots.txt Endpoint**: Created domain-aware robots.txt endpoint
  - Serves different robots.txt content based on requesting domain
  - Custom domain (`finan.eu.com`): Returns `Allow: /` with sitemap reference
  - Pages.dev domain: Returns `Disallow: /` blocking all crawlers
  - Unknown domains: Defaults to `Disallow: /` for security
  - Includes `X-Robots-Tag` header for additional crawler control

- **Domain Detection in Layout**: Added automatic domain detection logic
  - Detects when site is accessed via `.pages.dev` domains
  - Automatically applies `noindex, nofollow` meta tags for pages.dev access
  - Maintains `index, follow` for custom domain
  - Provides defense-in-depth SEO protection

## Changed

- **Layout Component**: Updated SEO meta tag handling
  - Modified `noindex` and `nofollow` props to default to domain-based detection
  - Pages accessed via pages.dev automatically get `noindex, nofollow`
  - Pages accessed via custom domain continue normal indexing behavior
  - Canonical URLs continue pointing to `finan.eu.com` (unchanged)

## Removed

- **Static robots.txt**: Deleted `/public/robots.txt`
  - Replaced with dynamic endpoint at `/src/pages/robots.txt.ts`
  - Enables domain-specific robots.txt content delivery
  - Prevents static file from overriding dynamic endpoint behavior

## Technical Details

### Multi-Layered SEO Strategy

This implementation uses a comprehensive three-layer defense approach:

1. **Layer 1 - CDN Edge Redirects**: `_redirects` file handles 99% of traffic at Cloudflare's edge, providing fastest and most effective solution
2. **Layer 2 - Dynamic robots.txt**: Blocks crawlers that somehow bypass redirects with domain-specific crawler instructions
3. **Layer 3 - Runtime Meta Tags**: Final safety net ensures even if crawlers ignore everything else, they see noindex tags

### Expected Impact

- **Immediate**: All pages.dev traffic redirects to custom domain
- **Week 2-4**: Google Search Console shows pages.dev URLs as "Excluded by redirect"
- **Week 4-8**: Pages.dev domain fully de-indexed from Google search results
- **Long-term**: Only custom domain appears in all search engine results
