# Changelog - 2025-12-04 18:12 (Europe/Helsinki)

## Changed

### EventSched Component Accordion - Improved Responsiveness

Refactored the accordion animation system in `src/components/EventSched.astro` to use a modern CSS Grid approach instead of JavaScript-calculated `max-height` values.

**Problem Fixed:**
- Accordion content was being cut off on smaller screens
- Content height calculations were static and didn't adapt when viewport size changed
- Required complex JavaScript with resize listeners to recalculate heights

**New Implementation:**
- Uses CSS Grid `grid-template-rows` for animation
- Transitions between `0fr` (collapsed) and `1fr` (expanded)
- Content automatically adjusts to any screen size without calculations
- No resize listeners needed

**How It Works:**

1. **Grid-based Animation:**
   ```css
   .event-content {
     display: grid;
     grid-template-rows: 0fr; /* collapsed */
     transition: grid-template-rows 0.3s ease-in-out;
   }
   ```

2. **Inner Wrapper with Overflow:**
   ```html
   <div class="event-content grid">
     <div class="overflow-hidden">
       <!-- actual content -->
     </div>
   </div>
   ```

3. **JavaScript Toggle:**
   ```javascript
   // Collapsed state
   content.style.gridTemplateRows = '0fr';

   // Expanded state
   content.style.gridTemplateRows = '1fr';
   ```

**Key Benefits:**
- ✅ Content never gets cut off on any screen size
- ✅ Automatically adapts to content reflow (text wrapping, etc.)
- ✅ Simpler JavaScript with no height calculations
- ✅ No resize event listeners needed
- ✅ Smooth CSS-based animations
- ✅ More performant and maintainable

**Technical Details:**
- `0fr` = grid row takes up 0 fraction of available space (collapsed)
- `1fr` = grid row takes up 1 fraction of available space (expands to fit content)
- Inner `overflow-hidden` div clips content when grid row is `0fr`
- CSS transitions the grid row size smoothly
- Content inside naturally determines the final height

This modern CSS technique is more reliable and eliminates the common accordion cut-off issues that occur with fixed `max-height` calculations.
