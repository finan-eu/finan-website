- **Improved blog card interaction**: Made entire blog cards clickable instead of just "Read More" link

## Fixed

### TypeScript Type Safety for Ghost CMS Data

- **Fixed nullable type handling**: Updated helper functions to properly handle nullable Ghost CMS fields
  - Updated `truncateExcerpt()` function to accept `string | null | undefined` parameter type
  - Updated `formatDate()` function to accept `string | null | undefined` parameter type
  - Added fallback values for null/undefined cases:
    - `truncateExcerpt()` returns empty string `''` if text is null/undefined
    - `formatDate()` returns `'Date unavailable'` if date is null/undefined
  - Resolved TypeScript compilation errors for `post.excerpt` and `post.published_at` fields

#### Technical Details

**Issue**: TypeScript compiler error when passing potentially nullable Ghost CMS fields to functions expecting strict `string` type.

**Solution**: Updated function signatures to accept nullable types and added defensive checks with appropriate fallback values.
